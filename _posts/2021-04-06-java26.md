# 추상화 클래스

우리가 알고있는 클래스는 보통 하나의 프로그램 이라고 이해할 수 있으면서<br>
무언가 동작하려는 설계도 같은 존재라고 쉽게 터득할 수 있습니다<br>
그렇다면 클래스에 추상화를 입힌다면 ```추상화 클래스``` 를 만들 수 있습니다<br>
그럼 도대체 이걸 어떻게 받아들여야 하는것 일까요?<br>

예를들어 A 클래스 B 클래스 C 클래스가 있다고 가정하고 각 클래스 안에는 멤버변수 & 멤버함수가 있을 것 입니다<br>
※ 추상클래스는 A B C 클래스들 간에 ```비슷한 필드와 메소드를 공통적으로 추출해 만들어진 클래스를 뜻합니다 ```<br>

마우스 라는 클래스를 구현해야 하는데 제조사는 여러개가 있다고 가정 해 봅시다<br>
A 회사 B 회사 C 회사는 각 제조사만의 스타일이 있으며 이것을 설계하고 제작하여 소비자들에게 판매를 합니다<br>

### A 회사
* 적당한 크기의 마우스를 제작 = > 클릭할때 소음은 커도 상관없습니다

### B 회사
* 아담하고 작은 마우스를 제작 = > 클릭할때 소음은 작게 나왔으면 합니다

### C 회사
* USB연결선이 없어도 되는 일명 무선마우스를 제작 = > 클릭할때 소음은 최대한 작게 나왔으면 합니다

여기서 각 회사마다 제조하는 마우스 의 공통점은 마우스의 ``` 클릭할때 나오는 소음``` 이고<br>
클릭 하면서 나오는 소음은 곧 동작하는 메소드가 공통적이다 라고 보면 됩니다<br>

※ 공통적인 메소드를 추출하고 추상클래스 안에 적용할 수 있습니다<br>

다른예로 들어보면 3개의 연필이 있는데 각 연필마다 색상이 다르게 나오고 모형 또한 다릅니다<br>

### A연필
* 겉표면 색상 : 검정색<br>
* 겉표면 모형 : 사각형<br>
* 연필심      : 남색<br>

### B연필
* 겉표면 색상 : 초록색<br>
* 겉표면 모형 : 육각형<br>
* 연필심      : 검정색<br>

### C연필
* 겉표면 색상 : 주황색<br>
* 겉표면 모형 : 삼각형<br>
* 연필심      : 노란색<br>

색상 , 모형 , 연필심 이건 공통적인 변수를 뜻하며 이 변수들을 추상클래스에 넣어 적용시킬 수 있습니다<br>
즉 추상화 클래스는 실체 클래스의 공통적인 부분을 추출하여 어느정도 규격을 잡아놓은 것을 말합니다<br>
그래서 실체클래스는 ```실제 객체를 생성할 정도의 구체성을 가지지만 ``` 추상클래스는 메소드와 내용물이 추상적이라<br>
객체를 생성할 수 없다는 특징을 가지고 있습니다<br>

### 1차 정리

* (1) 객체를 직접 생성할 수 있는 클래스는 실체클래스<br>
* (2) 실체클래스 의 공통적인 특성을 추출해서 선언한 클래스는 추상클래스 입니다<br>
※ 추출하는 것 : 변수 와 메소드<br>
* (3) 추상클래스는 객체를 생성할 수 없습니다 구현부가 없고 구체적이지 않다는 특징을 가집니다<br>
* (4) 추상클래스 < = > 실체클래스 는 상속적인 관계를 가집니다<br>

# 추상클래스 용도

추상클래스를 사용하는 이유는 3가지 이유가 있습니다<br>

* (1) 공통된 필드와 메소드를 통일하는 목적으로 사용합니다

3명의 개발자에게 선풍기를 상속받아서 각자 실체클래스를 구현해야하는 상황이 왔습니다<br>
이들의 필드명 과 메소드명은 제각각 다른 이름을 가지고 우여곡절 끝에 선풍기 클래스가 구현이 되었는데<br>
갑자기 선풍기 가 아닌 온풍기 클래스를 만들라고 하면 그 순간 유지보수는 커녕<br>
필드와 메소드를 전부다 체크하여 변경하고 아예 새롭게 개발한다는 느낌으로 가야할 것 입니다<br>

그래서 이런 상황을 대비할 수 있는 클래스가 바로 추상클래스 인데<br>
추상클래스 안에  ```미리 정의한 필드와 메소드를 가졌다면 ``` 실체클래스는 추상클래스를 상속받으며<br>
구현할 수 있는데 이렇게 되면 필드와 메소드 명의 통일성 과 동시에 유지보수성을 높일 수 있는<br>
장점을 가질 수 있게됩니다<br>


* (2) 실체클래스 구현할때 시간절약이 됩니다
